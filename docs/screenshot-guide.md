# Screenshot Guide

Use only synthetic, staged, or fully redacted screenshots for a public case study.

## Approved Screenshot Subjects

- Role-based menu with fake user data.
- Fleet dashboard with placeholder counts.
- LTO dashboard with placeholder status cards.
- Synthetic map view with fake route points.
- Route playback controls with fake route data.
- Violation dashboard with placeholder chart values.
- Black-points detail using fake driver/vehicle labels.
- LTO initiation form with placeholder text.
- Vehicle inspection form with blank or demo values.
- Driver assessment form with blank or demo values.
- Reports list with fake report rows.
- Tracker field request using demo labels.
- Quiz session using non-confidential training text.
- Profile management list using fake profile records.

## Required Redactions

Blur or replace:

- Names.
- Employee IDs.
- Vehicle IDs.
- Registration numbers.
- License numbers.
- Identity numbers.
- Phone numbers.
- Emails.
- Business groups.
- Regions.
- Customer names.
- Real addresses and location text.
- GPS coordinates.
- Route lines from real trips.
- Timestamps tied to real movements.
- Notification tokens.
- API responses.
- Document contents.

## Map Screenshot Rules

- Use fake coordinates only.
- Use synthetic routes only.
- Avoid recognizable production routes.
- Avoid map labels that reveal operating locations.
- Avoid real callout values.
